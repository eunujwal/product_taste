# Product Taste Lab

A Claude Code skill for systematically developing strong, portable product taste and product sense through structured exercises, critique, and reflection.

## What is this?

Product Taste Lab is an interactive coaching skill that turns vague "product sense" into a concrete, trainable craft. It runs structured sessions, assigns exercises, critiques answers, and helps product managers build their own coherent taste over time.

## How to use

This repository is a Claude Code skill. To use it:

1. Add this repository as a skill in your Claude Code configuration
2. Start a session by telling Claude you want to work on product taste
3. Claude will guide you through mode detection, calibration, and exercises

## Skill Structure

```
.claude/
  skills.md   — Full skill definition with exercises, phases, and session orchestration
  CLAUDE.md   — Style preset defining coaching voice, response structure, and grounding rules
```

## Core Philosophy

- **Taste Is Trainable** — Built through exposure, analysis, and feedback loops
- **Context Over Absolutes** — Everything evaluated for a specific user, problem, and objective
- **Show the Tradeoffs** — Clarity vs density, flexibility vs simplicity, familiarity vs novelty
- **Practice > Theory** — Hands-on teardowns, comparisons, and micro-redesigns over abstract advice
- **Reflect and Codify** — Build your own taste "playbook" of reusable principles
- **Cross-Domain Exposure** — Range beyond apps into physical products, services, and media
- **Outcome-Oriented** — Predict impact on users and metrics

## Modes

| Mode | Description |
|------|-------------|
| **Ongoing Training** | Systematically build product taste over weeks with a 4-week plan |
| **Interview Prep** | Short, intensive drills for structured thinking and communication |
| **Targeted Weakness** | Custom plan focused on a specific gap (e.g., problem framing, visual taste) |
| **Diagnostic** | Snapshot assessment of current taste and gap recommendations |

## Exercises

- **A: Single-Product Teardown** — Analyze what a product does, for whom, and how well
- **B: Comparative Taste Call** — Make relative quality judgments between alternatives
- **C: User Needs & Problem Reframing** — Focus on problems, not features
- **D: Micro-Redesign Under Constraints** — Surgical improvements with realistic limits
- **E: Taste Principles Extraction** — Turn opinions into reusable, explicit principles
- **F: Prediction & Outcome Loop** — Tie taste to outcome prediction and learning

## Taste Axes

Progress is tracked across six dimensions:

1. **User Understanding** — Empathy for jobs-to-be-done
2. **Problem Framing** — Sharp, solution-free problem statements
3. **Solution Fit** — Features mapped to problems
4. **Experience Quality** — Clarity, coherence, emotional resonance
5. **Tradeoff Awareness** — Reasoning through constraints and alternatives
6. **Outcome Orientation** — Linking choices to metrics and behaviors
